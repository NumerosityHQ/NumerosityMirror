This project can be used as a starting point to create your own Vaadin application with Spring Boot. It contains all the necessary configuration and some placeholder files to get you started.

The best way to create your own project based on this starter is \href{https://start.vaadin.com/}{\texttt{ start.\+vaadin.\+com}} -\/ you can get only the necessary parts and choose the package naming you want to use.\hypertarget{md__r_e_a_d_m_e_autotoc_md1}{}\doxysection{\texorpdfstring{Running the Application}{Running the Application}}\label{md__r_e_a_d_m_e_autotoc_md1}
There are two ways to run the application \+: using {\ttfamily mvn spring-\/boot\+:run} or by running the {\ttfamily Application} class directly from your IDE.

You can use any IDE of your preference,but we suggest Eclipse or Intellij IDEA. Below are the configuration details to start the project using a {\ttfamily spring-\/boot\+:run} command. Both Eclipse and Intellij IDEA are covered.\hypertarget{md__r_e_a_d_m_e_autotoc_md2}{}\doxysubsubsection{\texorpdfstring{Eclipse}{Eclipse}}\label{md__r_e_a_d_m_e_autotoc_md2}

\begin{DoxyItemize}
\item Right click on a project folder and select {\ttfamily Run As} -\/-\/\texorpdfstring{$>$}{>} {\ttfamily Maven build..} . After that a configuration window is opened.
\item In the window set the value of the {\bfseries{Goals}} field to {\ttfamily spring-\/boot\+:run}
\item You can optionally select {\ttfamily Skip tests} checkbox
\item All the other settings can be left to default
\end{DoxyItemize}

Once configurations are set clicking {\ttfamily Run} will start the application\hypertarget{md__r_e_a_d_m_e_autotoc_md3}{}\doxysubsubsection{\texorpdfstring{Intellij IDEA}{Intellij IDEA}}\label{md__r_e_a_d_m_e_autotoc_md3}

\begin{DoxyItemize}
\item On the right side of the window, select Maven -\/-\/\texorpdfstring{$>$}{>} Plugins-\/-\/\texorpdfstring{$>$}{>} {\ttfamily spring-\/boot} -\/-\/\texorpdfstring{$>$}{>} {\ttfamily spring-\/boot\+:run} goal
\item Optionally, you can disable tests by clicking on a {\ttfamily Skip Tests mode} blue button.
\end{DoxyItemize}

Clicking on the green run button will start the application.

After the application has started, you can view your it at \href{http://localhost:8080/}{\texttt{ http\+://localhost\+:8080/}} in your browser.

If you want to run the application locally in the production mode, use {\ttfamily spring-\/boot\+:run -\/Pproduction} command instead. \hypertarget{md__r_e_a_d_m_e_autotoc_md4}{}\doxysubsection{\texorpdfstring{Running Integration Tests}{Running Integration Tests}}\label{md__r_e_a_d_m_e_autotoc_md4}
Integration tests are implemented using \href{https://vaadin.com/testbench}{\texttt{ Vaadin Test\+Bench}}. The tests take a few minutes to run and are therefore included in a separate Maven profile. We recommend running tests with a production build to minimize the chance of development time toolchains affecting test stability. To run the tests using Google Chrome, execute

{\ttfamily mvn verify -\/Pit,production}

and make sure you have a valid Test\+Bench license installed.

Profile {\ttfamily it} adds the following parameters to run integration tests\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{-\/Dwebdriver.chrome.driver=path\_to\_driver}
\DoxyCodeLine{-\/Dcom.vaadin.testbench.Parameters.runLocally=chrome}

\end{DoxyCode}


If you would like to run a separate test make sure you have added these parameters to VM Options of JUnit run configuration\hypertarget{md__r_e_a_d_m_e_autotoc_md5}{}\doxysubsection{\texorpdfstring{Live Reload (optional)}{Live Reload (optional)}}\label{md__r_e_a_d_m_e_autotoc_md5}
With live reload, you can see the results of your code changes immediately. When you edit your Java code and recompile it, the application changes will be automatically reloaded and the browser is refreshed. This is done by leveraging \href{https://docs.spring.io/spring-boot/docs/2.1.5.RELEASE/reference/html/using-boot-devtools.html}{\texttt{ Spring Boot Developer Tools}}. To be able to see the changes in the browser tab, the page still needs to be reloaded. That can also be automated via a Live\+Reload browser extension. One such extension for Google Chrome is \href{https://chrome.google.com/webstore/detail/livereload/jnihajbhpnppcggbcgedagnkighmdlei}{\texttt{ Live\+Reload}}. In Firefox, \href{https://addons.mozilla.org/en-US/firefox/addon/livereload-web-extension/}{\texttt{ Live\+Reload -\/ Web extension}} can be used. You can find such similar extensions for other major browsers too. These extensions add an icon to your browser next to the address bar. To enable the extension, you should click that icon after you opened your application.

You can find more information at \href{https://vaadin.com/docs/flow/workflow/tutorial-spring-boot-live-reload.html}{\texttt{ Live Reload in Spring Boot Applications}} document.\hypertarget{md__r_e_a_d_m_e_autotoc_md6}{}\doxysection{\texorpdfstring{Structure}{Structure}}\label{md__r_e_a_d_m_e_autotoc_md6}
